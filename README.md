## Прогнозирование кредитного риска (Credit Risk Prediction)

### Описание проекта

Данный проект посвящен построению модели машинного обучения для оценки кредитного риска клиентов банка. Задача формулируется как бинарная классификация: на основе анкетных данных необходимо предсказать, относится ли клиент к категории высокого риска (вероятность невозврата кредита).

**Целевая переменная:** `Risk` (good — надежный клиент, bad — проблемный заемщик)

### Датасет

Используется модифицированная версия классического датасета **German Credit Dataset** с платформы Kaggle. Оригинальные данные подготовлены профессором Хофманном и доступны в репозитории UCI.

**Характеристики данных:**
- 1000 записей
- 10 признаков (социально-демографические и финансовые характеристики)
- Целевая переменная: good/bad (примерно 70%/30% — дисбаланс классов)

**Признаки:**
- `Age` — возраст клиента
- `Sex` — пол (male/female)
- `Job` — уровень квалификации (0-3)
- `Housing` — тип жилья (own/rent/free)
- `Saving accounts` — наличие сбережений (little/moderate/quite rich/rich)
- `Checking account` — состояние текущего счета (little/moderate/rich)
- `Credit amount` — сумма кредита
- `Duration` — срок кредита (месяцы)
- `Purpose` — цель кредита (car, radio/TV, education и др.)
- `Risk` — целевая переменная (good/bad)

### Этапы работы

#### 1. Разведочный анализ данных (EDA)
- Анализ пропусков и их заполнение категорией 'unknown'
- Визуализация распределений признаков
- Выявление ключевых факторов риска
- WoE/IV анализ для оценки предсказательной силы признаков

**Ключевые выводы EDA:**
- Финансовые признаки (Checking account, Duration, Saving accounts) — наиболее сильные предикторы
- Пропуски в данных неслучайны: категория 'unknown' соответствует низкому риску
- Цель кредита и тип жилья также значимы, особенно в комбинации

#### 2. Подготовка данных
- Создание бинарных версий числовых признаков (age_bin, amount_bin, duration_bin)
- One-hot кодирование категориальных признаков
- Стандартизация числовых признаков
- Разделение на train/val/test (85%/7.5%/7.5%) со стратификацией

#### 3. Моделирование
Сравнение пяти алгоритмов машинного обучения:
- Logistic Regression (baseline)
- Random Forest
- XGBoost
- LightGBM
- CatBoost

**Метрики оценки:**
- ROC-AUC — общее качество разделения классов
- F-beta (β=2) — целевая метрика с приоритетом полноты (recall)
- Recall для класса риска — доля выявленных проблемных клиентов

#### 4. Оптимизация гиперпараметров
Для каждой модели проведен RandomizedSearchCV с оптимизацией по F-beta метрике.

#### 5. Интерпретация модели
- Feature importance анализ
- SHAP-анализ для объяснения отдельных предсказаний
- Сравнение с выводами EDA

### Результаты

**Финальное сравнение моделей на тестовой выборке:**

| Модель | ROC-AUC | F-beta (β=2) | Recall (класс 1) |
|--------|---------|--------------|------------------|
| **Random Forest** | **0.789** | **0.725** | 0.909 |
| CatBoost | 0.781 | 0.724 | **1.000** |
| Logistic Regression | 0.786 | 0.720 | 0.818 |
| XGBoost | 0.777 | 0.691 | 0.773 |
| LightGBM | 0.763 | 0.662 | 0.818 |

**Лучшая модель:** Random Forest (после оптимизации гиперпараметров)

**Финальные метрики на тесте:**
```
              precision    recall  f1-score   support
           0       0.92      0.43      0.59        53
           1       0.40      0.91      0.56        22
    accuracy                           0.57        75
```

### Ключевые выводы

1. **Random Forest** показал наилучший баланс качества и интерпретируемости, выявляя 91% проблемных заемщиков.

2. **Финансовые признаки** (Checking account, Duration, Saving accounts) — ключевые факторы риска, что подтверждено EDA, важностью признаков и SHAP-анализом.

3. **Пропуски в данных информативны** — категория 'unknown' соответствует низкому риску и сохранена как отдельный уровень.

4. **Биннинг числовых признаков** улучшил интерпретируемость и выявил нелинейные зависимости.

5. **SHAP-анализ** подтвердил, что модель принимает решения, согласованные с бизнес-логикой.

### Интерпретация для бизнеса

Модель ориентирована на **максимальный охват рисковых клиентов** (recall = 0.91), даже ценой большого числа ложных срабатываний (precision = 0.40). Это полностью соответствует задаче кредитного скоринга, где пропустить дефолтный кредит обходится дороже, чем ошибочно отказать надежному заемщику.

**Практическая значимость:**
- Автоматизация принятия решений по кредитным заявкам
- Снижение финансовых потерь от невозвратов
- Возможность объяснить решение клиенту или регулятору (интерпретируемость)
